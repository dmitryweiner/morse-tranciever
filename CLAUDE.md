# CLAUDE.md — карта проекта для агента

Morse Transceiver: браузерный приёмопередатчик морзянки (TypeScript + Vite +
Web Audio) с обучающей диаграммой-деревом как на латунной карточке. Два
режима: Transmit (ключ Straight — длительностью нажатия, или Paddle — кнопки
точки/тире с автогенерацией длительностей; тон в динамики) и Receive
(непрерывный анализ микрофона → декодирование). Инфраструктура — по образцу
../formula-synth. Спецификация для людей — README.md. UI-тексты — на
английском (публичное приложение).

## Команды

```bash
npm run check      # tsc --noEmit && eslint . && vitest run — ОДНОЙ командой
                   # после любой правки (заменяет три отдельные)
npm test           # vitest (чистые src/morse/* и src/analysis/*) — зелёные
npm run lint       # eslint; `as`-касты ЗАПРЕЩЕНЫ (assertionStyle: never)
npm run dev        # Vite на :5173
npm run build      # tsc + сборка в ./docs (GitHub Pages)
npm run shot       # playwright-скриншоты в ./shots + смоук передачи: мышью
                   # «выстукивает» A на straight-ключе и T тапом по paddle.
                   # Флаги: --mobile (420×850), --preview (прод-сборка :4173)
npm run rx         # сквозной тест ПРИЁМА: генерирует WAV с морзянкой, играет
                   # его в фейковый микрофон Chromium и сверяет декод. Флаги:
                   # --msg "TEXT" --wpm N --hz N --preview; --file rec.wav
                   # [--expect TEXT] [--seed WPM] — реальная запись как
                   # микрофон. Финальная проверка цепочки mic→FFT→gate→decoder
npm run wav -- f.wav [--seed 5] [--hop 5] [--trace]  # ОФЛАЙН-стенд:
                   # разбор WAV, доминирующая частота, фронты гейта, декод
                   # через РЕАЛЬНЫЕ EnvelopeGate+AdaptiveDecoder. Основной
                   # инструмент отладки приёма на реальных записях — быстрее
                   # rx и без артефактов фейкового микрофона (startup-beep).
                   # --trace: таймлайн решений декодера (каждый элемент/буква
                   # со временем и текущими unit/ratio) — смотреть, ПОЧЕМУ
                   # метка классифицировалась не так
npm run gen -- --msg "SOS" [--wpm 15] [--hz 600] [--sr 16000] [--out f.wav]
                   # генератор тестовых WAV реальными модулями (TextSender +
                   # renderToneWav): синтетика с точными таймингами для
                   # ручной проверки Upload/приёма и отладки декодера
```

## Карта модулей

```
src/morse/code.ts     таблица A–Z + цифры + знаки .,?/=+-@ (ITU-R M.1677),
                      decodeCode, тип MorseEvent — единый формат событий
                      кеера И декодера (element/letter/word), UI рендерит их
                      одинаково. TREE_DEPTH=4 (глубина дерева, только A–Z) ≠
                      MAX_CODE_LENGTH (фактический максимум, 6): длинные коды
                      подсвечивают путь до 4-го уровня, дальше — только в
                      строке кода (решение пользователя — дерево не менять)
src/morse/timing.ts   PARIS-тайминги: unitMs(wpm)=1200/wpm; пороги
                      классификации — середины интервалов (2u и 5u)
src/morse/keyer.ts    TX-автомат (straight key): keyDown/keyUp/tick(t) →
                      MorseEvent[]; время параметром — тестируется скриптово
src/morse/paddle.ts   PaddleKeyer (электронный ключ): кнопки задают элемент,
                      длительности генерит автомат по WPM; удержание —
                      автоповтор, обе кнопки — чередование (ямбический),
                      память на 1 элемент (тап в паузу не теряется).
                      ВНУТРИ — обычный Keyer с синтетическими keyDown/keyUp:
                      паузы/буквы/события общие со straight-режимом
src/morse/sender.ts   TextSender (передача по тексту): выстукивает строку
                      точными PARIS-таймингами (паузы: элемент 1u, буква 3u,
                      слово 7u; неизвестные символы = пробел). Как paddle —
                      ВНУТРИ обычный Keyer с синтетическими keyDown/keyUp.
                      API: start(text,t)→events, stop(), tick(t), isToneOn,
                      isSending, currentIndex (подсветка буквы в UI).
                      Кончает посылку по коммиту последней буквы (хвостовую
                      словесную паузу не досиживает, trailing space нет)
src/morse/decoder.ts  RX-декодер с АВТО-адаптацией скорости И отношения
                      тире/точки: окно меток режется на 2 кластера по макс.
                      отн. скачку в отсортированном окне, порог = геом. среднее
                      СРЕДНИХ кластеров (устойчивее min/max к выбросам).
                      Разбиение валидно при разрыве средних >2.2, либо ≥1.5
                      при ≥2 метках в КАЖДОМ кластере и скачке ≥1.4 (сжатые
                      пищалки ~1.9:1 делятся, дрожь руки — нет). ratio =
                      avgDash/avgDot: EMA 0.25, кламп [1.5,4], дефолт 3,
                      учится только из окон с ≥2 метками в обоих кластерах;
                      setUnitMs сбрасывает к 3. Неоднозначное окно (один тип
                      меток) — дрейф к чтению (dur или dur/ratio), ближайшему
                      к оценке; порог точка/тире = unit×√ratio. Без ratio
                      серия тире утаскивала юнит к dur/3 и следующие точки
                      читались тире (тест «OOO EEE» на 1.9:1). Защиты (все —
                      из реальных записей пищалки, см. тест «bouncy beeper»):
                      • помеха = метка >8 юнитов ИЛИ >1200 мс (тире на 3 WPM);
                        классифицируется тире, но НЕ адаптирует и НЕ в окне
                      • кластер точек пере-оценивает юнит только из ≥2 меток
                        (одиночный осколок 105 мс утаскивал оценку с 240)
                      • анти-дребезг: разрыв тона <25 мс (абсолютно) — не
                        пауза, НО склейка разрешена только к метке, уже
                        тянущей на элемент (≥0.45 юнита): дребезг — мерцающий
                        хвост большой метки; град осколков 5–45 мс «возни»
                        (виден на хопе 5 мс) иначе сшивался в псевдо-метку
                        точечного масштаба и рожал ложную букву (тест «hail
                        of tiny shards», samples/TEST*.wav);
                        метка <0.45 юнита — осколок «возни»/дребезга:
                        не элемент, не адаптирует, НЕ сдвигает таймер пауз
                        (порог поднимался 0.35→0.45 по записям samples/);
                        пока метка не финализирована, паузы не считаются
                        (иначе буква коммитится поверх метки)
                      Диапазон трекинга ~±2×, дальше — re-seed слайдером
src/morse/envelope.ts EnvelopeGate: уровни дБ → on/off; адаптивные пол/пик
                      (быстро вниз/вверх соответственно), гистерезис 65/50%
                      (порог ВКЛ поднят с 50: тихая «возня» ключом сидит у
                      середины размаха), якорь по первому кадру, MIN_SNR 12 дБ.
                      SignalGate = EnvelopeGate И тональность И захват несущей:
                      контраст пика над медианой полосы (вкл >12 дБ / выкл <7)
                      ПЛЮС стабильность частоты (±80 Гц/кадр; нужна и для
                      удержания, иначе случайное совпадение защёлкивает шум).
                      Захват несущей (carrierHz, показывается в rxInfo): замок
                      после ~120 мс подряд стабильных кадров тона (короткие
                      осколки «возни» не успевают — их пик у пьезо бывает на
                      ГАРМОНИКЕ, ранний замок резал первое тире!); чужая
                      частота — не сигнал, пока не продержится ~0.4 с
                      (перезахват); ~20 с тишины — замок отпускается.
                      ВСЕ выдержки — в МИЛЛИСЕКУНДАХ, конструкторы принимают
                      hopMs (дефолт 15 — эталон калибровки REF_HOP_MS) и
                      пересчитывают в кадры; кадровые EMA (пол/пик/несущая)
                      приводятся к эквиваленту по времени 1-(1-α)^(hop/15) —
                      иначе хоп 5 мс утраивает посекундную адаптацию и пол
                      съедает тире 3 WPM (тест «does not starve»)
src/audio/tone.ts     Sidetone: осциллятор всегда работает, гейтится gain'ом
                      с рампой 4 мс (иначе щелчки). ensure() звать СИНХРОННО

src/audio/mic.ts      MicAnalyser: getUserMedia (echo/noise/AGC выключены —
```

## Ключевые решения (не ломать)

- **DSP отделён от Web Audio**: src/morse/* не знают про DOM/AudioContext —
  их гоняет vitest. Всё аудио — тонкие обёртки, проверяются npm run rx/shot.
- **Кеер и декодер эмитят одинаковые MorseEvent** — дерево/текст/буква
  обновляются одним applyEvents, режимы не дублируют UI-логику.
- **Пороги классификации — середины интервалов** (точка/тире — 2u, буква —
  2u, слово — 5u пауз): прощают кривую ручную манипуляцию.
- **Junk-guard декодера (8u / 1200 мс)** появился не теоретически: фейковый
  микрофон Chromium на старте выдаёт «бип» ~1.5 с, который травил адаптацию
  (сейчас его отсекает ещё и тональность SignalGate) — см. rx.mjs.
- **rx.mjs: полный Chromium (channel: 'chromium')** — headless-shell не
  умеет getUserMedia («NotSupportedError»). Разрешение — через
  context.grantPermissions(['microphone']), НЕ --use-fake-ui (промпт висит).
  В WAV — 4 с стартовой тишины: захваченное до старта сессии Chromium отдаёт
  одним спрессованным всплеском. Не открывать getUserMedia до клика — файл
  начинает играть с первым захватом. В --file-режиме фейковый захват ещё и
  ПОДКАЧИВАЕТ уровень (AGC-констрейнты не действуют): тихая возня на записи
  всплывает и даёт мусор в начале — эталон декода реальных записей всегда
  офлайн-стенд (npm run wav), браузерный --file — проверка проводки.
- **eslint запрещает `as`** — DOM через instanceof-хелперы (src/ui/dom.ts).
- **[hidden] { display:none !important }** в style.css — панели режимов
  скрываются атрибутом, а у них display:flex.
- iOS: unlock переключателя «без звука» (src/audio/iosUnlock.ts) зовётся
  синхронно в жесте из tone.ensure(); ключ — pointer events + setPointerCapture,
  touch-action:none (иначе long-press/скролл ломают манипуляцию).

## Бэклог

- Тренировочные режимы: приём сгенерированного текста, проверка ответа.
- Гашение цикла на скрытой вкладке в TX-режиме (в RX гасить нельзя — приём).
- Словарная/контекстная коррекция декодера — единственный путь дожать
  записи со слипшимися точками (samples/: SOS1, SOS2, TEST.wav — см.
  «Карту модулей»).
